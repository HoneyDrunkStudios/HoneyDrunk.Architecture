# Sensitive Inventory

**Purpose:** The canonical index of every credential, identifier, secret, and identity binding the Grid holds. Per ADR-0083 D2.

**Scope:** Names and metadata only — **never values**. Per invariant 8 (secrets never appear in logs/traces) and invariant 103 (ADR-0083 D7).

The `status: planned` rows (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`) use a `2099-01-01` far-future sentinel the drift workflow treats as no-escalation. `INITIATIVES_SYNC_TOKEN` and `GRID_HEALTH_PAT` were **discovered during this pass** — they were not in the ADR-0083 seed list; their `Use Cases` are inferred from the token names and should be verified.

**Maintenance:**
- New entries land via the standup-procedure onboarding hook per ADR-0083 D6 and ADR-0082 (`constitution/node-standup.md`).
- Existing entries' `Current Expiration` updates land at rotation time per the relevant walkthrough under `infrastructure/walkthroughs/`.
- The scheduled `external-credentials-check.yml` workflow (ADR-0083 D5, `HoneyDrunk.Actions`) parses this file and escalates at T-30 / T-7 / T+0.
- Schema integrity is enforced by that workflow's schema-check sub-step — table-format drift fails the workflow fast. **Do not reorder or rename columns.**
