# Universe — Power System (The Rulebook)

**Status:** Seed (living document; canon accretes — keep loose)
**Last updated:** 2026-06-20

The legible rulebook every in-universe game draws from. This is the **primary reusable node** of the whole universe: the power system's *rules* are immutable shared canon, even though each game expresses them with different combat *presentation* (PDR-0012 §C, "rules are canon; presentation varies").

## Design DNA (what makes it satisfying)

Inspired by JJK, HxH, and Naruto — but the throughline isn't aesthetic, it's that **power is legible and constrained**:

1. **Hard rules and limits.** Power can be reasoned about. (Nen-style vows trade restriction for strength; techniques have innate conditions.)
2. **Costs and tradeoffs.** Power is never free.
3. **Outthinking beats out-stat-ing.** The weak can beat the strong by exploiting the rules. *This is the secret to making "starts weak" fun:* being weak isn't low numbers — it's *not having figured out the system yet*. Every fight won by cleverness is a hit of the core fantasy.
4. **Constraints breed identity.** A specific, limited technique is more memorable than generic strength.

> If players end up talking about *the system* — the combos, the counters, the rule-exploits — across every game in the universe, the power system is doing its job.

## Structure: typed essence

Power is grounded in **essence** (`cosmology.md`), which is **plural/typed**:

- A character's **identity = which essence types they have and acquire.** This is what makes the customizable, starts-weak protagonist meaningful — the build *is* the character.
- Types imply **affinities and interactions** (rock-paper-scissors-style relationships — to be designed, not locked).
- **Acquisition is the engine** — the whole world (and the player's progression) is beings obtaining types they lack, by earning / taking / being granted (see `cosmology.md` acquisition paths; keep the *earning/climb* path primary).

## Reuse boundary (rules vs. presentation)

| Reusable canon (the node) | Per-game variable |
|---|---|
| Essence types & their interactions | Action *or* turn-based combat (**no hybrid**) |
| Costs, conditions, limits, vows | Pacing, controls, UI |
| Acquisition rules | Which subset of the system a given game exposes |

The reusable artifact is the **ability rules/data**; the per-game **combat driver** reads those rules. Game 2 (the power-system node) is where this rulebook first becomes concrete code — built for *that* game, extracted for reuse only when game 3 needs it.

## Open questions (resolve as games force them)

- The number, names, and interaction graph of essence types.
- The "vow/restriction → power" mechanic: present? how costed?
- How acquisition feels moment-to-moment (learning, looting, pact rituals).
- Action vs. turn-based for the *first in-universe* combat game (game 2) — explicitly a thing game 2 prototypes; the operator is open to either and dislikes hybrid; different games may differ.
