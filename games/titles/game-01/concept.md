# Title — Game 1: Learning Project (codename TBD)

**Status:** Planned (target start ~January 2027 — PDR-0012)
**In-universe:** No (off-canon by design)
**Last updated:** 2026-06-20

## Purpose

The first game. Its **only** job: take the operator through the **full loop once** — Unity → build → polish → Steam store page → ship → observe real players. Optimized for *finishing and learning*, not revenue, not lore.

## Hard constraints

- **Tiny, finishable scope.** Weeks-to-a-couple-months of part-time work. Single core mechanic.
- **Off-canon and disposable by design.** Deliberately *not* in the shared universe — we don't want a clumsy first attempt locking canon we'll regret, and we want full freedom to throw work away.
- **PC / Steam.** (See `../../roadmap.md` setup checklist.)
- **Lean on free/cheap asset packs + AI** so art/audio skill never blocks finishing.

## Not yet decided (the next design conversation)

Game 1's actual *concept* is intentionally open — it's the next thing to shape. Candidate directions for a "good first game" (a single tight mechanic):
- A score-chaser / arcade loop
- A small puzzler
- A tiny self-contained action loop

## Definition of done

- Ships on Steam (or a deliberate decision to ship on itch.io instead).
- A real store page exists.
- At least a handful of real players have played it and given feedback.
- The operator has touched every stage of the pipeline at least once.

> Lessons learned here feed game 2 (the first in-universe title and the power-system node). See `../../roadmap.md`.
