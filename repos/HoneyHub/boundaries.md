# HoneyHub — Boundaries

## What HoneyHub Owns
- Architecture decisions (ADRs)
- Grid topology catalogs (Nodes, packages, relationships, signals)
- Agent routing rules
- Issue template definitions
- Per-repo context documentation
- Generated artifact output

## What HoneyHub Does NOT Own
- **Code** — No application or library code
- **CI workflows** — Belong in HoneyDrunk.Actions
- **Repo-internal documentation** — Each repo has its own docs/
- **Implementation details** — Only high-level boundaries and integration points
