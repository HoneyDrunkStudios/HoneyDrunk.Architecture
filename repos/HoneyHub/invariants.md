# HoneyHub — Invariants

1. **Catalogs must reflect actual state.** If a Node is added, removed, or versioned, update catalogs.
2. **Routing rules must be consistent with catalogs.** Keywords in `repo-discovery-rules.md` must match entries in `nodes.json`.
3. **Issue templates must be complete.** Every template includes frontmatter, acceptance criteria, and label suggestions.
4. **Generated artifacts are ephemeral.** Contents of `/generated/` can be deleted at any time.
