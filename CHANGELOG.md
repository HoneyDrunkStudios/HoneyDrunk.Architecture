- **Added a local-first carve-out to invariant 45** (`constitution/invariants.md`). Invariant 45 forbids hardcoded rates / routing policies / capability declarations, requiring App-Configuration sourcing. The carve-out permits a local-first, offline-capable app with no hosted backend at runtime (HoneyHub) to ship a **frozen, read-only snapshot** of those values, on narrow conditions: it is generated at build/publish time as a one-way projection of the App-Config canonical surface (never hand-authored/coined), App Config stays the single source of truth, and snapshot-derived values are tagged `fidelity: derived` so staleness is visible (ADR-0092 D2). A service with a live config path must still source directly and may not bundle to dodge the live dependency. Unblocks the `honeyhub-v1` routing engine (HoneyHub PR #33) against its invariant-45 Grid finding; aligns the constitution with the ADR-0092 2026-06-08 synced-snapshot amendment and ADR-0090/ADR-0091 D4 local-first posture. Also committed the scope doc for that work — `generated/work-items/active/honeyhub-v1/11-honeyhub-routing-engine-synced-snapshot.md` — so the PR carries a real work-item link (invariant 32), and captured the honeyhub-v1 as-built record (`generated/work-items/active/honeyhub-v1/implementation-notes.md`, implementer-authored per the ADR-0008 amendment / Arch PR #555). The as-built record is a forward-looking draft and currently lists routing (HoneyHub PR #33) as shipped ahead of its merge.
- Fixed the ADR-0086 Grid Agent Runner so the **runner fetches the PR diff and inlines it into the review prompt** for both the Codex and Claude passes, instead of instructing each agent to fetch the diff from GitHub itself. The Claude pass runs with host credentials scrubbed and in read-only `--permission-mode plan` with no diff-capable tool, so it could not retrieve a private PR diff and silently returned no findings, while Codex's read-only sandbox could still execute the fetch — masking the gap and effectively making every Grid review single-perspective. This realigns the implementation with ADR-0086 D2's context-loading contract ("the PR diff … is a property of the agent prompt, not the transport") and the safety posture that the runner "passes only GitHub diff/context … to child agents": a new `New-ReviewDiffSection` helper pulls the unified diff with the runner's GitHub App token (new optional `Accept` override on `Invoke-GitHubApi`), caps it at 200k characters, wraps it in explicit untrusted-content markers, and embeds it in the prompt — so the agents need no network access or credentials. Engine-only change in `lib/Queue.psm1` + `lib/GitHub.psm1`; takes effect on the live runner only after an operator redeploy into the installed `TrustedRunnerRoot`.
