## Ranked Opportunities

### 1. HoneyHub BYOK Cloud Execution and Loop Console

- **Thesis:** For individual developers and solo operators already using local and cloud coding agents, HoneyHub should prove its launch/readiness path and validate BYOK cloud execution before any broader Dev-surface commercial work.
- **Buyer:** Individual developer or AI-heavy solo operator who uses Codex, Claude Code, Copilot, or GitHub cloud agents across multiple repos and wants mobile monitoring, session control, and spend-aware routing.
- **Smallest viable wedge:** Distribution 90 launch checkpoint, public v0.1.0 package, and one BYOK waitlist page with concrete price and reserve action.
- **Build cost:** Already in the active queue; no incremental opportunistic packet reduces the work.
- **Opportunity cost:** Any new packet delays public release and turns the lane back into substrate exploration.
- **Kill criteria:** BYOK waitlist misses its threshold; Loop Console scope expands beyond the minimal operator loop surface; cloud execution would require subscription auth instead of BYOK API keys.
- **Why HoneyDrunk:** HoneyHub already dogfoods the exact agent-runner, PR-as-artifact, local-first, usage-governance workflow it sells.

### 2. NovOutbox First-Beta Commercial Trial

- **Thesis:** For indie .NET SaaS developers and small .NET teams, NovOutbox remains the best commercial pressure test for the Grid, but only if the next slice is narrowed to the first beta tenant.
- **Buyer:** Indie .NET SaaS developer or 2-to-5-person .NET team that needs transactional email/SMS orchestration without hand-rolled provider glue.
- **Smallest viable wedge:** Go/slip decision, private-repo automation proof, wrapper scaffold path, and first-beta dependency cut for tenant/API/billing basics.
- **Build cost:** Already committed as current-focus work; opportunistic scope would add risk.
- **Opportunity cost:** Expanding NovOutbox now competes with HoneyHub's release loop and increases pre-signal commercial surface.
- **Kill criteria:** Multi-tenant wrapper compromises internal Notify use; first-beta scope cannot be kept to tenant auth, API key issuance, minimal billing, and delivery handoff.
- **Why HoneyDrunk:** Notify, Communications, Vault, Data, Transport, Web.Rest, Auth, Payments, and Pulse all get production pressure through one product trial.

### 3. Curiosities Phase 0 Content Spike

- **Thesis:** For curious city walkers, a curated one-district discovery set can test whether the "question marks become places" loop is worth physical action before mobile build begins.
- **Buyer:** Local explorer or traveler who likes hidden landmarks, public art, history, walking tours, and collectible place memories.
- **Smallest viable wedge:** One dense launch district, source/licensing checklist, safety rubric, and about 25 reviewed POIs.
- **Build cost:** About 1-2 solo-dev weeks for a strict content spike; much more if it expands into app work.
- **Opportunity cost:** Pulls attention from HoneyHub and NovOutbox, so it should stay a bounded risk test.
- **Kill criteria:** Reviewed POIs cannot be produced at quality, safety, and pace; testers browse but do not walk to unlocks.
- **Why HoneyDrunk:** The studio's Lore/Knowledge patterns and aesthetic can make place discovery feel authored instead of directory-like.

### 4. Agent Action Ledger / Audit Evidence Wrapper

- **Thesis:** For regulated .NET shops adopting agentic workflows, an append-only action ledger remains a strong future product, but not before NovOutbox and Audit prove the required substrate.
- **Buyer:** Engineering lead at a 20-to-80-person regulated .NET SaaS company that needs attributable evidence for model calls, tool calls, approvals, cost, and outcomes.
- **Smallest viable wedge:** Deferred; future `HoneyDrunk.Audit.Cloud` or equivalent ingest surface only after PDR-0010 gates clear.
- **Build cost:** PDR-0010 estimates about 8 solo-dev weeks after Audit Phase 1 and commercial signal; currently higher because prerequisites are not mature.
- **Opportunity cost:** Opens a second commercial front before the first has signal.
- **Kill criteria:** Notify/NovOutbox signal does not transfer to .NET-shop buyer demand; Audit Phase 1 does not mature on the PDR-0010 schedule; buyers do not pay above the threshold after launch.
- **Why HoneyDrunk:** The Grid's Audit substrate is intentionally distinct from telemetry, giving the future product a cleaner record-of-truth boundary than generic AI observability.
